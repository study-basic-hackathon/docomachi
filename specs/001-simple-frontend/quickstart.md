# Quickstart: 001-simple-frontend の動作確認

**Feature**: 一問フロントエンド（麻雀手牌クイズ）  
**Date**: 2025-02-28

実装完了後、以下でサンドボックス起動・初期データ投入・画面確認まで行う。

---

## 前提

- Node.js 20 以上
- AWS CLI および Amplify 環境のセットアップ済み（既存プロジェクトの手順に従う）
- `.env` に必要な環境変数（例: `GITHUB_TOKEN`）が設定されていること

---

## 1. 依存関係のインストール

```bash
npm install
```

---

## 2. サンドボックス（Amplify バックエンド）の起動

```bash
npx ampx sandbox --profile <your-profile>
```

またはプロジェクトで定められたサンドボックス起動コマンドを実行する。起動完了後、`amplify_outputs.json` が更新されていることを確認する。

---

## 3. 初期データの投入

`doc/seed` のデータを投入する。プロジェクトで用意されている seed スクリプトを実行する。

例（スクリプトが `scripts/seedMahjongHands.ts` などの場合）:

```bash
npx tsx scripts/seedMahjongHands.ts
```

または `doc/seed/mahjong_hands.json` を読み込む別の手順があればそれに従う。  
投入後、DynamoDB（MahjongHand テーブル）にレコードが存在することを確認する。

---

## 4. フロントエンドの起動

```bash
npm run dev
```

ブラウザで `http://localhost:3000` を開く。

---

## 5. 画面確認の手順

1. **トップページ**: 緑背景・スタートボタンが表示されていること。
2. **スタートクリック**: 出題画面（`/quiz` 等）に遷移すること。
3. **出題表示**: 手牌が萬・索・筒・字の順で牌画像として表示されていること。解答用に全種類の牌が選択可能であること。
4. **解答**: 1枚以上牌を選択すると「解答する」が有効になること。正解の組み合わせで送信すると正解モーダル、不正解で送信すると不正解モーダルが表示されること。
5. **モーダル閉じた後**: 出題画面に留まり、やり直し or 戻るボタンでトップへ戻れること。
6. **エラー時**: 出題取得に失敗した場合、エラーメッセージとリトライ・トップへ戻るが表示されること（必要に応じて認証未ログイン等で再現）。

---

## 6. テストの実行

```bash
npm run test
```

Jest でミニマムなテストが通ることを確認する。

---
