**Date**: 2025-03-22

## Overview

本機能は **リポジトリ内のドキュメントとバイナリ画像** のみを扱い、永続化アプリケーションスキーマや API ペイロードは新設しない。

## エンティティ

### 1. プロジェクト README（`README.md`）

| 属性 | 説明 | 制約・バリデーション |
|------|------|----------------------|
| 配置 | リポジトリルート | ファイル名は `README.md`（GitHub の既定入口） |
| 言語 | 日本語 | FR-006: 見出し・本文・箇条書きの説明部は日本語。固有名詞の英語は可。英語のみの段落・英語主体の並記は不可。 |
| 節: アプリ概要 | 利用者・提供価値 | FR-001 |
| 節: 開発プロセス | Speckit の利用 | FR-002 |
| 節: インフラ / バックエンド | AWS Amplify Gen 2 | FR-003 |
| 節: 技術スタック | 主要 FW・パッケージ | FR-004、`package.json` との整合 |
| 節: 画面 | スクリーンショット参照 | FR-005、少なくとも 1 画像 |

**関係**: 画像エンティティを相対パスで参照する。

### 2. スクリーンショット（画像ファイル）

| 属性 | 説明 | 制約・バリデーション |
|------|------|----------------------|
| 配置（推奨） | `doc/readme/screenshots/` | research の決定。PNG または WebP 等、GitHub が表示可能な形式。 |
| 内容 | ユーザー向け主要 UI | 個人情報・秘密情報を含まない（仕様 Edge Cases）。 |
| 件数 | 1 枚以上 | FR-005 |

**関係**: ルート `README.md` から 1 対多で参照可能（仕様上は最低 1 枚で可）。
