# Data Model: 007-frontend-design

**Feature**: Screen Layout and Visual Polish  
**Date**: 2025-03-07

## Overview

本機能は **表示・レイアウト・UX の変更のみ** であり、新規の永続化データや API スキーマは不要です。既存のドメイン概念（手牌、待ち牌、問題、解答）は変更しません。

## 参照する既存エンティティ

| 概念 | 説明 | 本機能での利用 |
|------|------|----------------|
| トップページ | アプリのエントリ画面 | バナー画像表示・ボタン配置・ローディング |
| 待ち牌 / 牌選択 | クイズの解答候補（TileCode） | 色別グループ・改行・選択強調・スクロール |
| クイズ問題 | 手牌・正解待ち牌 | 既存のまま。表示レイアウトのみ調整 |

## UI 状態（クライアントのみ）

実装時に必要なクライアント側の状態は次のとおり。永続化・API は不要。

| 状態 | 用途 |
|------|------|
| ローディング表示中か | オーバーレイの表示/非表示 |
| トーストメッセージ | エラー・タイムアウト時の一過性表示（Sonner が内部管理） |
| 選択中の待ち牌 | 既存の `selectedTiles`。表示は border 等で強調 |
| ビューポート / 向き | スマホ時の横固定やメディアクエリ用（必要に応じて） |

## バリデーション・制約

- トップバナー: 画像アセット `top-banner.png` を `public/` または `public/images/` から参照する（仕様で指定済み）。
- 待ち牌のグループ: 既存の `TileCode` を萬子・索子・筒子・字牌の 4 グループに分けて表示する。順序は既存の `TILES_DISPLAY_ORDER` に準拠。

## 結論

新規のデータモデル・API 契約は不要。既存の型（`TileCode`, `QuestionItem` 等）と UI 状態のみで実装可能。
