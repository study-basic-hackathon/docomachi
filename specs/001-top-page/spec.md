## User Scenarios & Testing *(mandatory)*

### User Story 1 - トップページの表示とスタート操作 (Priority: P1)

初回訪問者がアプリのトップページを開くと、麻雀を連想させる緑の背景が表示され、上部にバナー画像、目立つ位置にスタートボタンが表示される。ユーザーはスタートボタンを押して次の画面へ進める。

**Why this priority**: トップページは入口であり、第一印象と初回操作を担うため最優先。

**Independent Test**: トップページを表示し、緑背景・バナー・スタートボタンの存在を確認し、スタート操作で次へ進めることを検証すれば価値を確認できる。

**Acceptance Scenarios**:

1. **Given** ユーザーがアプリのルートにアクセスした, **When** ページが読み込まれる, **Then** 麻雀をイメージさせる緑の背景が表示される
2. **Given** トップページが表示されている, **When** ユーザーが画面を確認する, **Then** 上段にバナー画像（top-banner.png）が表示される
3. **Given** トップページが表示されている, **When** ユーザーが確認する, **Then** スタートボタンが明確に表示され操作可能である
4. **Given** トップページが表示されている, **When** ユーザーがスタートボタンを押す, **Then** 次の画面または適切な遷移が行われる

---

### Edge Cases

- バナー画像が存在しない、または読み込みに失敗した場合でも、ページは表示されスタート操作は可能である（画像のみ非表示または代替表示）
- 画面幅が狭い場合でも、緑背景・バナー・スタートボタンが崩れずに表示される
- スタートボタン連打時は重複遷移や二重送信が発生しない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはトップページで麻雀を連想させる緑の背景を表示しなければならない
- **FR-002**: システムはトップページの上段にバナー画像（元の配置元: img/top-banner.png）を表示しなければならない
- **FR-003**: システムはトップページにスタートボタンを配置し、ユーザーが操作可能にしなければならない
- **FR-004**: スタートボタン操作時、システムは定義された次の画面またはフローへ遷移しなければならない
- **FR-005**: バナー画像はプロジェクト内の適切な場所に配置され、トップページから参照できる状態にしなければならない

### Key Entities *(include if feature involves data)*

- **トップページ**: アプリの入口画面。緑背景・バナー・スタートボタンで構成される単一の表示単位（永続データなし）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーがルートURLにアクセスしてから3秒以内にトップページの主要要素（背景・バナー・スタートボタン）が表示される
- **SC-002**: ユーザーはスタートボタン1回の操作で意図した次の画面へ遷移できる
- **SC-003**: 一般的なデスクトップおよびモバイルの画面幅で、レイアウトが崩れずに表示される

## Assumptions

- バナー画像（top-banner.png）はプロジェクト作成時に `img/` に存在し、本機能で「適切な場所」へ移動する運用を想定する
- 「スタート」押下後の遷移先は別仕様または既存フローに委ね、本仕様では「次の画面へ遷移する」ことのみを要求する
- 緑の背景は「麻雀をイメージさせる」程度のトーンとし、具体的な色コードやデザイン詳細は実装時に決定する
