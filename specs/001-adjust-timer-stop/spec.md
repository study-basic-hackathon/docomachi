# Feature Specification: 解答タイマー15秒化と停止

**Feature Branch**: `001-adjust-timer-stop`  
**Created**: 2026-03-21  
**Status**: Draft  
**Input**: User description: "15秒に変更してください。また、解答するのボタンをクリックしたらタイマーは止めてください"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 15秒以内に解答する (Priority: P1)

プレイヤーとして、問題表示から15秒以内に回答できるようにし、従来より余裕をもって判断したい。

**Why this priority**: 制限時間の変更はプレイ難易度と正答体験に直結し、この要望の中心機能であるため。

**Independent Test**: 問題表示後に15秒以内で回答するケースと、15秒経過して未回答のケースを確認し、前者は通常判定、後者は不正解になることを確認できる。

**Acceptance Scenarios**:

1. **Given** 問題が表示されタイマーが開始している, **When** プレイヤーが15秒以内に回答する, **Then** その回答は通常どおり正誤判定される
2. **Given** 問題表示後に未回答のまま15秒経過する, **When** 制限時間に到達する, **Then** その問題は自動で不正解として確定する

---

### User Story 2 - 解答操作でタイマーを止める (Priority: P2)

プレイヤーとして、「解答する」ボタンを押した時点でタイマー進行を停止し、送信後に残り時間が変化しないことを期待する。

**Why this priority**: 解答送信後に時間が進み続けるとユーザーの認知と結果表示の整合が崩れるため、体験の一貫性確保が重要。

**Independent Test**: 残り時間がある状態で「解答する」を押し、モーダル表示中に残り時間表示が更新されないことを確認できる。

**Acceptance Scenarios**:

1. **Given** タイマーが進行中である, **When** プレイヤーが「解答する」を押す, **Then** タイマーは即時停止し以後のカウントダウンは行われない
2. **Given** 「解答する」を押して結果が確定した, **When** 結果モーダルを表示中である, **Then** 残り時間は送信時点の値のまま変化しない

---

### User Story 3 - 既存の時間切れ体験を維持する (Priority: P3)

プレイヤーとして、制限時間が15秒に変わっても時間切れ時の不正解体験はこれまでと同じ挙動で受けたい。

**Why this priority**: 既存フローが変わると学習コストが増えるため、仕様変更点を時間長と停止条件に限定するため。

**Independent Test**: 未回答で15秒経過させ、既存の不正解モーダルが表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 問題表示後に未回答のまま15秒経過する, **When** 時間切れ処理が発火する, **Then** 通常の不正解時と同じモーダルが表示される

### Edge Cases

- 「解答する」押下と制限時間到達が同時刻に発生した場合は、時間切れとして不正解を優先すること
- 「解答する」を連打してもタイマー停止処理と結果確定処理が重複しないこと
- 次の問題へ遷移した際は前問の停止状態を引き継がず、15秒タイマーを新規開始すること

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、各問題の表示開始と同時に15秒の解答制限時間を開始しなければならない
- **FR-002**: システムは、問題表示から15秒経過時点で未回答の場合、自動的に不正解として確定しなければならない
- **FR-003**: システムは、プレイヤーが「解答する」を押した時点で当該問題のタイマー進行を停止しなければならない
- **FR-004**: システムは、「解答する」押下後の結果表示中に残り時間表示を更新してはならない
- **FR-005**: システムは、残り時間を画面上に表示し、残り3秒以下で赤文字表示に切り替えなければならない
- **FR-006**: システムは、時間切れ時に通常不正解時と同一の不正解モーダルを表示しなければならない
- **FR-007**: システムは、同一問題でタイマー停止処理と結果確定処理を複数回実行してはならない

### Key Entities *(include if feature involves data)*

- **問題タイマー状態**: 1問に紐づく時間管理情報。制限時間（15秒）、残り秒数、進行状態（進行中/停止）を持つ
- **解答確定イベント**: 解答ボタン押下または時間切れで発生する問題確定情報。確定理由（手動回答/時間切れ）と結果状態を持つ
- **結果モーダル表示状態**: 問題確定後に表示される結果UI状態。モーダル種別（正解/不正解）と表示中フラグを持つ

## Assumptions

- 15秒制限は全問題共通で適用し、問題種類ごとの例外は設けない
- タイマー停止対象は「解答する」押下後の現在問題のみとし、次問題開始時には新規に開始する
- 残り3秒以下の赤文字ルールは既存仕様を継続する

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーテストで、参加者の95%以上が制限時間を15秒として認識できる
- **SC-002**: 「解答する」押下後のセッションの100%で、残り時間表示が停止し変化しない
- **SC-003**: 未回答で15秒経過したケースの100%で、通常不正解と同一のモーダルが表示される
- **SC-004**: 解答押下と時間切れ競合ケースで、結果確定が1回のみ行われるケースが100%である
